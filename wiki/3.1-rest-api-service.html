Purpose and Scope

The REST API Service is the client-facing entry point for the Kafka Request-Reply system. It exposes HTTP endpoints that accept calculation requests, forwards them to Kafka for processing using a synchronous request-reply pattern, and returns responses to clients. This component demonstrates how to implement synchronous communication over Kafka's asynchronous messaging infrastructure.

For information about the listener services that process the requests, see Listener Service and Listener1 Service.

  • README.md

Component Architecture

The REST API Service is implemented as a Spring Boot application within the restapi directory. It uses Spring Web for REST capabilities and Spring Kafka for Kafka integration, with a focus on the ReplyingKafkaTemplate component that enables synchronous request-reply messaging over Kafka.

Key Components

The REST API Service consists of these primary components:

ComponentTypeDescription
REST ControllerSpring ControllerHandles HTTP requests, initiates Kafka messaging, returns HTTP responses
ReplyingKafkaTemplateSpring KafkaSpecialized Kafka template that enables synchronous request-reply patterns
NotificationDispatchRequestJava RecordData model containing calculation parameters (base, exponent)
NotificationDispatchResponseJava RecordData model containing calculation result

Message Flow

The following diagram illustrates the message flow initiated by the REST API Service:

The full flow sequence:

  1. Client sends HTTP POST to the /calculate endpoint with base and exponent values
  2. Controller converts the request to a NotificationDispatchRequest object
  3. ReplyingKafkaTemplate adds correlation headers and sends to request topic
  4. First Listener service processes the request (Math.pow) and forwards to intermediate topic
  5. Second Listener service processes the intermediate result (Math.sqrt) and sends to reply topic
  6. ReplyingKafkaTemplate receives the response and matches it to the original request
  7. Controller converts the response to HTTP format and returns to client

ReplyingKafkaTemplate Usage

The ReplyingKafkaTemplate is the key component that enables synchronous request-reply patterns over Kafka:

The template performs these critical functions:

  1. Generates a unique correlation ID for each request
  2. Creates a dynamic reply topic
  3. Adds these as headers to the outgoing message
  4. Waits for a response matching the correlation ID
  5. Returns the response once received or times out if no response arrives

API Endpoint

The REST API Service exposes a single HTTP endpoint:

POST /calculate

Accepts a calculation request with base and exponent values, processes it through Kafka, and returns the final calculation result.

Request Format:

{
  "base": 2.0,
  "exponent": 10.0
}

Response Format:

{
  "result": 32.0
}

Processing Steps:

  1. Converts request to NotificationDispatchRequest
  2. Sends to first listener via Kafka
  3. First listener calculates Math.pow(base, exponent) (e.g., 2^10 = 1024)
  4. Second listener calculates Math.sqrt(result) (e.g., √1024 = 32)
  5. Returns final result to client

Running the Service

To run the REST API Service:

  1. Ensure Kafka/Redpanda is running (using the provided docker-compose.yml)
  2. Navigate to the restapi directory
  3. Execute mvn spring-boot:run
  4. The service will be available at http://localhost:8080

Test the service with curl:

curl -X POST http://localhost:8080/calculate -H "Content-Type: application/json" -d '{"base": 2.0, "exponent": 10.0}'

Expected response:

{"result": 32.0}
